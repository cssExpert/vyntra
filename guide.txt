# Vyntra Services Guide

## Database Management (pgAdmin)

Because this project uses PostgreSQL, we use pgAdmin instead of phpMyAdmin to manage the database.

### 1. How to start the service
Open your terminal in the `infrastructure` folder and run:
`docker-compose up -d`
This will start the database and the pgAdmin interface in the background.

### 2. How to access pgAdmin
Open your web browser and go to:
http://localhost:5050

### 3. How to log in
Use the following default credentials:
- Email: admin@vyntra.com
- Password: admin

### 4. How to connect pgAdmin to your Database
Once logged in, pgAdmin needs to be told how to connect to the database container.
1. In the "Quick Links" section on the dashboard, click **Add New Server** (or right-click "Servers" in the left sidebar -> Register -> Server...).
2. In the **General** tab:
   - Name: Vyntra DB (or anything you prefer)
3. In the **Connection** tab, enter exactly these details:
   - Host name/address: `postgres` (This targets the docker container exactly by its service name)
   - Port: `5432`
   - Maintenance database: `vyntra_db`
   - Username: `vyntra`
   - Password: `vyntra_dev_password`
   - Save password: Check the box
4. Click **Save**.

### 5. How to view your Data
1. In the left sidebar, expand **Servers** -> **Vyntra DB**.
2. Navigate to: **Databases -> vyntra_db -> Schemas -> public -> Tables**.
3. You will see all your database tables here (e.g., pages, users).
4. To view the data, right-click a table -> **View/Edit Data** -> **All Rows**. This opens a spreadsheet-like view on the right side.

### 6. How to run Custom SQL Queries
1. Right-click on your database (`vyntra_db`) in the left sidebar.
2. Select **Query Tool**.
3. Type your SQL query (e.g., `SELECT * FROM pages;`) and click the "Play/Run" (triangle) button in the top toolbar.

## Running the Full Project with Docker

### Step 1 — Start infrastructure services (Postgres, Redis, MinIO, etc.)
Open your terminal in the project root and run:
`docker compose -f infrastructure/docker-compose.yml up -d`

This starts:
- PostgreSQL       → localhost:5432
- Redis            → localhost:6379
- MinIO (storage)  → localhost:9000 | console: localhost:9001
- Meilisearch      → localhost:7700
- pgAdmin          → localhost:5050

### Step 2 — Set up the API .env (first time only)
`cp apps/api/.env.example apps/api/.env`

The defaults already match the docker-compose credentials, so no edits needed for local dev.

### Step 3 — Build & run the API container
`docker build -t vyntra-api .`

`docker run --rm --network vyntra-network --env-file apps/api/.env -e DATABASE_URL="postgresql://vyntra:vyntra_dev_password@vyntra-postgres:5432/vyntra_db" -e REDIS_URL="redis://vyntra-redis:6379" -p 3001:3001 vyntra-api`

Note: Inside Docker, hostnames change from localhost to container names (vyntra-postgres, vyntra-redis),
so the -e overrides above are required.

### Step 4 — Run the web frontend (outside Docker, recommended for dev)
`cd apps/web && pnpm dev`

Web runs at http://localhost:3000
API runs at http://localhost:3001

### Stop all services
`docker compose -f infrastructure/docker-compose.yml down`

Add -v flag to also wipe data volumes:
`docker compose -f infrastructure/docker-compose.yml down -v`
